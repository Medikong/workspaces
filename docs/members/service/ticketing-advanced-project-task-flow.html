공연 티켓 서비스 기반 심화 프로젝트 Task 흐름

1. 전제

기존 의료 서비스 구현은 더 이상 유지하지 않고, 공연 티켓 발행 서비스로 도메인을 전환한다. 서비스 파일은 새 도메인 기준으로 재구성한다.

Kafka는 도메인 이벤트 스트림으로 유지한다. 서비스 간 즉시 조회는 REST, 후속 처리는 Kafka event로 분리한다.

2. 3인 역할

담당 역할 주요 책임
Person 1 Platform / Cloud / Runtime K8s, Kong/Istio 설치 기반, AWS VPC/RDS/S3, PDB/RBAC/ServiceAccount
Person 2 Service / Gateway / Mesh 서비스 API, Kafka event 발행/소비, Kong route, Istio traffic policy
Person 3 Quality / Observability / DevSecOps pytest/Newman/k6, Grafana/Loki/Tempo, Trivy/SonarQube, 요구사항 충족표
