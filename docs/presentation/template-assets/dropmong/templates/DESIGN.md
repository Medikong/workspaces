# DropMong Presentation System

이 문서는 DropMong 기술 발표 템플릿의 시각 규칙입니다. 개별 페이지는 이 규칙을 바탕으로 서로 다른 정보를 보여 주되, 같은 발표자료에 넣었을 때 하나의 체계로 보여야 합니다.

## 화면 규격

- 캔버스: 1600×900px, 16:9
- 안전 여백: 좌우 80px, 위 56px, 아래 48px
- 기본 구획: header 150px, body 620px, footer 56px
- 본문 중심 시각물 점유율: 화면의 60~75%
- 제목: 최대 2줄, 58~64px, 700~800 weight
- 본문: 22~30px, 보조 라벨 16~20px
- 수치: 40~64px, tabular numerals

## 글자와 문구

- Pretendard 계열 글꼴
- 제목과 짧은 라벨은 명사형 종결
- 설명은 짧고 자연스러운 한국어
- KEDA, HPA, Kafka, event, Queue lag, p99 같은 기술 용어는 원문 유지
- 번역체 표현, 명사 연속 나열, 가운데점 과다 사용 제외
- 화면 문구에 `LANGUAGE-POLICY.json`의 금지 표현과 동사형 종결 제외
- placeholder는 `발표 제목`, `발표자`, `소속`, `YYYY.MM.DD`, `000 ms`, `00.0%` 형식

## 선과 형태

- 기본 경계선 1px, 도식 연결선 2px, 핵심 연결선 3px
- 모서리 8px, 12px, 16px
- 그림자 없음, 필요한 경우 한 곳에만 매우 약한 그림자
- 독립 카드 최대 3개
- 수치 4개는 카드 4장이 아닌 하나의 수치 스트립
- 화살표는 동기 호출, 비동기 event, 관측 연결, 제어 연결을 선 모양과 색으로 구분

## Header, Body, Footer

- Header: 왼쪽 DropMong 로고, 작은 분류 라벨, 제목
- Body: 한 페이지에 하나의 중심 도식, 차트, 표, 코드 또는 결론
- Footer: 프로젝트명, 발표자명, 날짜 자리
- 출처 영역과 페이지 번호 제외
- 마스코트: 표지, 장 전환, 마무리 페이지에만 사용

## 페이지군

- Title: 표지와 장 전환, 큰 제목과 제한적인 마스코트
- Body: 공통 header와 footer, 하나의 중심 시각물
- End: 확인 내용과 다음 작업, 작은 마스코트
- Appendix: 필요한 경우 Body 규칙을 그대로 적용

## 데이터 규칙

- 실제 결과가 아닌 페이지: `예시 데이터 · 실제 측정값 아님` 또는 `설정 예시 · 운영값 아님` 표시
- 수치 자리: `000 RPS`, `000 ms`, `00.0%`, `00m`, `HH:MM:SS`
- 이전 HPA 전용 페이지: decision→Ready 약 11~14초, concert 1→4, 실패율 34.64%, p99 약 14.9초만 사용
- 이전 HPA 수치에 `이전 HPA 실험 · 새 실험 결과 아님` 표시
- 새 실험 결과, 개선율, 우열 결론을 임의로 생성하거나 암시하지 않음

## 로고와 마스코트

- 로고는 원본 비율을 유지하고 좌측 상단 기준으로 배치
- 마스코트는 원본 캐릭터 형태를 유지하고 새 표정이나 의상 생성 제외
- 본문 정보보다 큰 마스코트 제외
- 상품 UI, 쇼핑 장면, 홍보용 장식 제외

## 금지 구성

- 다크 테마, 네온, 글로우, 강한 그라데이션
- 축소 페이지 모음과 컴포넌트 카탈로그
- 대시보드처럼 여러 차트를 한 화면에 배치한 구성
- 같은 크기의 카드 4개 이상
- 설명이 없는 장식 아이콘
- 출처 footer, 페이지 번호, 워터마크
- 레퍼런스 이미지의 직접 삽입, 잘라 붙이기, 원본 레이아웃 복제
