이전 결과와 새 실험을 한 화면에서 구분

수치, 관찰 조건, 현재 상태를 함께 표시

06 · Research Question

청중이 기억할 한 가지 질문

대상, 조건, 비교군, 측정값을 문장 안에 포함

[대상]에서 [조건]을 같게 둘 때 [방법 A]와 [방법 B]의 [측정값]은 얼마나 다른가?

꼭 넣을 내용

  • 대상
  • 같은 조건
  • 바꿀 값
  • 비교할 수치

07 · Hypothesis

예상과 다른 가능성을 같은 크기로 제시

결과를 보기 전에 틀렸다고 볼 조건까지 작성

예상하는 결과

방법 A의 회복 시간이 방법 B보다 짧은 결과

차이가 없을 가능성

두 방법의 회복 시간과 자원 사용량이 비슷한 결과

예상이 틀렸다고 볼 조건

더 느린 회복, 같은 품질, 더 큰 자원 사용량

입력: expected, alternative, checks결과 확인 전 작성

08 · Comparison

같은 조건과 바꿀 값을 좌우로 구분

한 번에 두 묶음만 비교

같이 둘 조건

  • 이미지와 설정
  • 입력 데이터
  • 자원 제한
  • 실행 시간
구분

바꿀 값

  • 방법 A의 설정
  • 방법 B의 설정
  • 결과 수치

09 · System Model

입력에서 결과까지 각 구성요소의 역할 표시

직접 조절하는 값과 단순히 측정하는 값을 구분

입력요청 또는 이벤트
제어기설정과 결정
처리 대상파드 또는 워커
결과품질과 효율

역할 표시

  • 조절 대상
  • 관측 신호
  • 외부 의존성
  • 이번 비교의 제외 범위

10 · Traffic Chart

입력의 시간 변화와 단위를 함께 표시

실측, 가상 입력, 예상값 가운데 하나를 명시

차트에 필요한 설명

  • x축 시간
  • y축 단위
  • 입력이 바뀌는 시점
  • 데이터의 현재 상태

11 · Experiment Design

앞 단계가 통과해야 다음 단계로 이동

원인이 섞이지 않도록 한 단계에서 한 가지를 변경

  1. 측정 준비
    필요한 신호가 같은 시간으로 수집되는지 확인
  2. 단일 대상 기준선
    처리 한계와 첫 품질 저하 확인
  3. 고정 크기 비교
    대상을 늘렸을 때 처리량이 늘어나는지 확인
  4. 방법 비교
    같은 입력에서 선택한 방법을 반복 실행

고정 조건

데이터, 이미지, 자원, 외부 의존성, 실행 시간

12 · Experiment Matrix

비교군마다 바뀌는 값과 그대로 둘 값 정리

아직 정하지 못한 값은 숫자를 만들지 않고 표시

비교군바꿀 값같은 조건상태
기준선고정 수량이미지와 입력준비 완료
방법 A설정 A이미지와 입력결정 필요
방법 B설정 B이미지와 입력결정 필요

실험 전 · 결과 값 없음

13 · Metric / Statistics

시간, 품질, 효율을 따로 재고 함께 비교

평균 하나로 긴 지연과 실패를 숨기지 않는 구성

주요 수치

  • 시작부터 회복까지 걸린 시간
  • p99와 오류율
  • 처리량
  • 성공 작업당 자원 사용량

계산과 반복

ΔT = trecovery − tstart
  • 조건별 최소 3회 준비 실행
  • 중앙값과 분산 확인
  • 본 반복 수는 준비 실행 뒤 결정

14 · Limitation

이번에 확인할 범위와 나중에 볼 범위 구분

아직 실행하지 않은 결과를 결론처럼 쓰지 않는 화면

이번 범위

  • 한 환경
  • 한 가지 기본 입력
  • 준비된 노드
  • scale-to-zero 제외

나중에 확인

  • 순간 증가
  • 반복 peak
  • 노드 자동 확장
  • 다른 저장소

15 · Conclusion

지금 알 수 있는 답과 바로 할 일을 함께 정리

확인한 사실과 앞으로 얻을 결과를 구분

지금 확인한 내용 → 먼저 준비할 항목 → 같은 조건의 비교 → 결과에 따른 다음 행동

마무리 항목

  • 한 문장 답
  • 첫 실행
  • 아직 정하지 못한 값
  • 후속 확인
