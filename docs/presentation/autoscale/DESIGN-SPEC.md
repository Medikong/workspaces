# 기술 실험 발표 디자인 명세

## 1. 시각 기준

이 명세는 다음 세 디자인 시트를 직접 확인한 결과를 공통 규칙으로 정리한 문서입니다.

- `assets/dropmong-system-architecture-sheet-color.png`
- `assets/dropmong-system-architecture-sheet.png`
- `assets/dropmong-ui-ux-component-sheet.png`

이미지는 발표에 넣지 않습니다. 화면 구성에 필요한 색, 선, 간격, 모서리와 정보 밀도만 참고합니다.

레퍼런스의 핵심은 흰 바탕, `#111827` 잉크색, `#6C3DF5` 보라색, `#F2F4F7` 구분선, 2px 아이콘 선, 8~16px 모서리입니다. 큰 그라데이션, 진한 그림자, 넓은 라벤더 면, 둥근 카드의 반복은 사용하지 않습니다.

## 2. 색 토큰

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `canvas` | `#FCFCFD` | 전체 배경 |
| `surface` | `#FFFFFF` | 카드와 표 |
| `surface-soft` | `#F8F7FF` | 한 화면의 보조 면 한 곳 |
| `ink` | `#111827` | 제목과 핵심 본문 |
| `ink-soft` | `#374151` | 설명 |
| `muted` | `#6B7280` | 상태와 보조 정보 |
| `line` | `#F2F4F7` | 카드와 표의 기본 선 |
| `line-strong` | `#E5E7EB` | 표 머리글과 큰 구분선 |
| `purple` | `#6C3DF5` | 핵심 경로와 선택 항목 |
| `purple-ink` | `#5425D6` | 밝은 배경 위 보라색 글자 |
| `lavender` | `#B69BFF` | 보조 강조 |
| `lavender-soft` | `#F5F2FF` | 선택된 영역의 옅은 배경 |
| `success` | `#15803D` | 기준 충족 후보 |
| `info` | `#2563EB` | 확인된 사실 |
| `warning` | `#B45309` | 결정 필요와 선행 조건 |
| `danger` | `#DC2626` | 오류와 SLO 기준 초과 |

의미색은 글자나 선에 우선 사용합니다. 넓은 면을 채울 때는 5% 안팎의 옅은 색만 사용합니다. 색만으로 상태를 구분하지 않고 `확인`, `실험 전`, `결정 필요` 같은 문구를 함께 둡니다.

## 3. 글자 규칙

- 글꼴: Pretendard
- 표지 제목: 58~64px, 700~800 weight, 최대 2줄
- 일반 제목: 42~48px, 700~800 weight, 최대 2줄
- 설명: 23~25px, 400~600 weight
- 본문: 21~23px, 행간 1.45~1.55
- 표와 카드: 19~22px
- 보조 라벨: 18px 이상
- 수치: tabular numerals

제목은 슬라이드의 결론이나 확인 대상을 짧은 명사구로 적습니다. 설명은 제목을 되풀이하지 않고 조건이나 범위를 명사형으로 보충합니다. 긴 문장 대신 대상, 조건, 비교 기준과 수치를 나눈 짧은 구를 사용합니다.

## 4. 간격 토큰

| 토큰 | 값 | 사용 위치 |
| --- | ---: | --- |
| `space-1` | 8px | 라벨 내부와 짧은 묶음 |
| `space-2` | 12px | 제목과 설명 사이 |
| `space-3` | 16px | 같은 내용 묶음 |
| `space-4` | 24px | 카드 내부와 목록 사이 |
| `space-5` | 32px | 머리글과 본문 사이 |
| `space-6` | 48px | 큰 구역 사이 |

1600×900 화면의 바깥 여백은 좌우 80px, 위 56px, 아래 48px입니다. 임의의 margin 대신 위 토큰을 사용합니다. 제목, 설명, 본문 사이는 각각 12px와 32px를 기본값으로 둡니다.

## 5. 카드와 표

- 한 화면의 독립 카드는 2~3개
- 카드 모서리 12px, 큰 패널 16px
- 카드 내부 여백 20~24px
- 기본 선 1px, 구조를 보여주는 도식 선 2px
- 기본 그림자 없음
- 겹침을 구분해야 할 때만 `0 2px 8px rgba(17, 24, 39, 0.05)`
- 내용이 짧은 카드는 내용 높이만 사용
- 표 머리글은 옅은 회색 또는 보라색 선으로 구분

카드가 많아지면 표, 한 줄 단계 목록, 시계열 또는 하나의 큰 도식으로 바꿉니다. 같은 모양의 카드 네 개를 반복하지 않습니다.

## 6. 레이아웃 구조

기본 슬라이드는 다음 세 구역을 순서대로 사용합니다.

1. 분류 라벨, 제목, 짧은 설명
2. 한 가지 질문을 설명하는 본문
3. 필요한 슬라이드에만 상태 또는 범위 표기

기본 레이아웃에는 절대 위치 footer가 없습니다. 출처 파일명과 절 번호는 발표자 노트와 마지막 참고자료에 둡니다. 수치의 상태가 중요할 때만 본문 안에 `이전 실험`, `실험 전` 라벨을 붙입니다.

## 7. 발표 동작

- Slidev `v-click`과 Reveal.js `fragment`는 실행 순서와 비교군 공개에만 사용
- 180~240ms의 짧은 fade 또는 8px 이하 이동
- 자동 재생, 반복 강조, bounce, 회전, 장식용 parallax 제외
- reduced-motion 환경에서 즉시 표시

## 8. 언어와 접근성

- 제목, 부제, 본문, 표와 카드 문구는 짧고 구체적인 명사형 종결
- 화면의 동사형 종결 금지, 발표자 노트의 자연스러운 `-습니다`체 허용
- 전문 용어는 첫 등장 때 쉬운 뜻을 함께 표시
- 표 머리글과 데이터 셀을 semantic element로 구분
- 일반 글자 대비 4.5:1 이상, 큰 글자 3:1 이상
- 화면 글자 18px 이상
- 차트와 도식에 축, 단위, 입력 상태 또는 대체 설명 제공

세부 문구 규칙은 `LANGUAGE-POLICY.json`에서 관리합니다.

## 9. 대표 화면 검수 기준

레퍼런스와 발표 화면을 나란히 놓고 다음 항목을 확인합니다.

| 항목 | 레퍼런스 기준 | 발표 화면 기준 |
| --- | --- | --- |
| 대표색 | `#6C3DF5` | 선택 경로와 핵심 수치에만 사용 |
| 배경 | 흰색 중심 | `#FCFCFD`와 흰 표면 |
| 선 | 얇고 선명한 구분 | 1px 카드 선, 2px 도식 선 |
| 모서리 | 8·12·16px | 카드 12px, 큰 패널 16px |
| 그림자 | 거의 없음 | 기본 없음 |
| 밀도 | 정렬된 작은 묶음 | 카드 2~3개, 빈 공간의 목적 확인 |
| 간격 | 반복되는 단위 | 8·12·16·24·32·48px 토큰 |

자동 검사는 overflow, 작은 글자, 대비, 머리글과 본문 간격, 과도한 빈 공간을 확인합니다. 최종 판단은 1600×900 캡처의 수동 비교로 마무리합니다.
