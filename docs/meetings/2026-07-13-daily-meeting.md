# 2026-07-13 데일리 미팅

## 목적

발표 구간별 주 담당자를 정하고, 역할별 기반 구조 작업과 도메인별 공통 책임 범위를 구분한다.

## 결정 사항

| 결정 | 이유 | 연결 문서 |
| --- | --- | --- |
| 역할 A, B, C는 프로젝트 전체에서 함께 사용할 기반 구조의 주 담당을 뜻한다. | 역할 구분을 특정 도메인의 개발 책임과 혼동하지 않기 위해서다. |  |
| 각 구성원은 역할별 기반 구조 작업과 별개로 자신이 맡은 도메인을 처음부터 끝까지 책임진다. | 공통 기반과 도메인 구현을 분리하지 않고 실제 서비스에 함께 적용하기 위해서다. |  |
| 문제 정의, 시장 분석, 전체 아키텍처, 성능 수치, 장애 대응 결과, 회고는 세 명이 공동으로 준비하고 발표한다. | 프로젝트 전체 맥락과 결과는 모든 구성원이 함께 설명할 수 있어야 하기 때문이다. |  |

## 역할별 기반 구조 담당

| 역할 | 담당자 | 기반 구조 책임 | 발표 구간 |
| --- | --- | --- | --- |
| 역할 A | 김정엽 | CI/CD, GitOps, 롤백 | CI/CD, GitOps, 롤백 |
| 역할 B | 최범휘 | Observability(관측성), SLO, 알림 | Observability, SLO, 알림 |
| 역할 C | 박명수 | 서비스 메시, 부하 테스트, 카오스 검증 | 서비스 메시, 부하 및 카오스 검증 |

## 공동 발표 구간

| 발표 구간 | 담당 |
| --- | --- |
| 문제 정의, 시장 분석, 전체 아키텍처 | 김정엽, 최범휘, 박명수 공동 |
| 성능 수치, 장애 대응 결과, 회고 | 김정엽, 최범휘, 박명수 공동 |

## 도메인별 공통 책임 범위

각 구성원은 자신이 맡은 도메인에서 다음 항목을 공통으로 책임진다.

- 도메인 규칙과 사용자 시나리오
- 백엔드 구현
- API 및 이벤트 계약
- 컨테이너 이미지
- Kubernetes 배포 선언
- 환경별 설정과 시크릿
- 메트릭, 로그, 추적
- 기능 테스트, 통합 테스트, 부하 테스트
- 장애 대응 런북
- 결과 자료와 발표 근거

## 2026-07-14 멘토링 준비

각 구성원은 내일 멘토링에서 자신이 맡은 도메인을 설명하고 피드백을 받을 수 있도록 소개 자료를 준비한다.

| 준비 내용 | 담당 | 완료 기준 |
| --- | --- | --- |
| 담당 도메인의 목적, 사용자 시나리오, 핵심 규칙 소개 | 김정엽, 최범휘, 박명수 | 도메인의 역할과 주요 사용 사례를 설명할 수 있다. |
| 백엔드 구현, API 및 이벤트 계약, 배포 구성 현황 정리 | 김정엽, 최범휘, 박명수 | 현재 구현 범위와 남은 작업을 구분해 설명할 수 있다. |
| 멘토에게 확인할 질문과 피드백이 필요한 지점 정리 | 김정엽, 최범휘, 박명수 | 도메인별 질문과 논의할 내용을 자료에 포함한다. |

## 후속 정리 필요

- 각 구성원이 맡은 도메인 이름과 경계를 문서에 연결한다.
- 역할별 기반 구조가 각 도메인에 적용됐는지 확인할 수 있는 증거 위치를 연결한다.
